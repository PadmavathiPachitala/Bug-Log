User Guide & Documentation

Learn how to maximize your productivity and manage your debugging knowledge base with BugLog.

🔑 01. Getting Started: Sign In

BugLog supports automatic user onboarding to make starting out frictionless.

  • Go to the Login Page.
  • Enter any email address and a password (minimum 6 characters).
  • If the email is new, the system will **automatically register it** and log you in immediately. If it exists, it will log you in normally.
💡 Example:
Type developer@gmail.com and password securepass123. If this email is not in the database, BugLog automatically registers it and redirects you to the dashboard.

📝 02. Logging a New Bug

Record bugs as you solve them to build your personal searchable index.

  • Click **+ New Bug** on the Dashboard.
  • Error Message / Stack Trace: Paste raw terminal or console error outputs.
  • Root Cause / Hypothesis: Summarize why it failed.
  • Verified Solution: Describe what you did to resolve it and paste any code snippets.
  • Takeaways: Detail what you'll tell your future self (great for technical interview prep!).
💡 Example Bug Log:
  • Title: useEffect Infinite Loop in React
  • Error Message: Maximum update depth exceeded. This can happen when a component repeatedly calls setState...
  • Root Cause: Forgot to add the dependency array [] as the second argument to useEffect, causing it to run on every state change.
  • Solution: Added , [dependencies]) at the end of the hook.

🔍 03. Search & Filters

Find any previously solved bug instantly to stop solving the same problems twice.

  • Use the search bar to query keywords inside title, stack trace, solutions, or tags.
  • Click **⚙️ Advanced Filters** to filter logs by specific Technology, Priority, AI Tool used, or Creation Dates.
  • Toggle the quick filter pills (*All, Open, Resolved, ★ Favorites, Critical*) for rapid filtering.
💡 Search Example:
To find database connection issues, type "MongoNetworkError" in the search bar. Use the ⚙️ Advanced Filters to limit the search to Node.js technology with Critical priority.

🤖 04. AI Prompt Vault

Save and organize the prompts that successfully debugged your code.

  • Navigate to the **Prompt Vault** from the navbar.
  • Save prompts with target AI tools (e.g., *Gemini, Claude*) and effectiveness ratings.
  • Click the copy icon on any prompt card to copy it to your clipboard instantly.
💡 Saved Prompt Example:
"I have the following Node.js stack trace [paste stack trace]. Identify the line causing the null reference and write a safe optional chaining fallback."

📊 05. Analytics & Exports

Track your learning progress and back up your logs.

  • Check the **Analytics** page to visualize trends, tech distributions, and heuristic AI diagnostics.
  • Click **Export Log** on the Dashboard to download your complete history in **CSV, Markdown, or PDF** formats.
💡 Export Formats:
* **CSV:** Raw data for importing into Excel or Google Sheets.
* **Markdown:** Beautifully formatted log entries ready for a GitHub wiki.
* **PDF:** Structured, print-ready diagnostic reports for team code reviews.
